# Security Guide

## Threat Model

The Revit MCP Server connects an AI client to a live Revit session. The threat surface is:

1. **Unauthorized tool invocation** — an attacker calls tools without a valid token.
2. **Destructive operations** — `delete_elements`, `bulk_update_parameter`, etc. modify the model.
3. **Data exfiltration** — project data is returned to the AI client.
4. **SSRF / host confusion** — the server is tricked into connecting to unexpected hosts.

## Recommended Configuration

### For local development (single user, not networked)

```env
REVIT_MCP_AUTH_MODE=api_key
REVIT_MCP_API_KEY=<auto-generated or any strong key>
```

The MCP server runs on stdio and is only accessible to processes on the same machine. This is the safest deployment model.

### For team / CI use (server exposed over SSE)

```env
REVIT_MCP_AUTH_MODE=jwt
REVIT_MCP_JWT_SECRET=<64-char random hex>
REVIT_MCP_JWT_EXPIRY_MINUTES=30
```

Issue tokens with short expiry and rotate the secret regularly.

**Never expose the SSE transport to the public internet** without a reverse proxy (nginx/Caddy) with TLS.

### Auth mode `none`

Only use `none` mode in an air-gapped local environment with no external network access. It is disabled by default in production deployments by intentional design.

## pyRevit Routes Security

The pyRevit Routes API does **not** have built-in authentication. By default it binds to `localhost:48884` and is therefore only accessible from the local machine. Do not modify this binding unless you fully control the network.

If you need remote access, tunnel over SSH rather than binding to `0.0.0.0`:

```bash
ssh -L 48884:localhost:48884 user@revit-machine
```

## Safe Operations

All write operations (create, modify, delete) run inside Revit `Transaction` objects. If an error occurs mid-transaction, Revit rolls back automatically.

For destructive batch operations (`delete_elements`, `bulk_update_parameter`), consider:
- Saving a Revit backup/snapshot before running
- Using Revit's built-in undo after validating results
- Filtering to a small subset first, then scaling up

## Reporting Vulnerabilities

Open a GitHub issue marked `[security]` or email the maintainer directly.
